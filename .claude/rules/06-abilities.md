# 7. Abilities and Effects

> Source: Star Wars Unlimited Comprehensive Rules v7.0 (3/6/26)
> See also: 05-keywords.md for Keyword Abilities (7.5)

---

## 7.1 General

- 7.1.1 An ability is specialized game text indicating how a card affects the game. If a card has printed abilities, they're in the text box. An upgrade can give a unit an ability via "gains" text — treat the unit as having the ability in its text box while the upgrade is attached.
- 7.1.2 Five ability types: action, constant, event, keyword, triggered.
- 7.1.3 An **effect** is a non-cost part of an ability that has the potential to change the game state. Some effects resolve separately from the ability that created them, or replace its standard resolution.
- 7.1.4 If an ability affects both players simultaneously and can be resolved simultaneously, resolve it that way. Otherwise, the controlling player chooses the order in which each player is affected.
- 7.1.5 Resolve the effects of an ability in the order they are written. If the ability lets a player choose options, those may be resolved in any order.
- 7.1.6 **Abilities that Modify Actions**: Some abilities instruct the player to take an action ("Play a card," "Attack with a unit," or "Use an Action Ability") with additional effects — this is a "modified action." A modified action is resolved as though the player took the corresponding action, with modifiers applied at the appropriate step.
  - a. Modified "Play a Card" may include restrictions on card type, cost adjustments, playing from non-hand zones, or other effects.
  - b. Modified "Attack With a Unit" may include attack restrictions, temporary modifiers/keywords, or other effects.
  - c. Modified "Use an Action Ability" may include restrictions on how the ability is used or other effects.
  - d. Abilities triggered during or as a result of a modified action are nested abilities. See 7.6.
- 7.1.7 If an ability instructs a player to take multiple actions (e.g., "Play three cards"), perform them sequentially. Nested abilities must be resolved before proceeding to the next action.

## 7.2 Action Abilities

- 7.2.1 Indicated by the bolded word **"Action."** Most have a cost in brackets. Using an action ability is one type of action a player can take. See 6.4.
- 7.2.2 A player must pay the full cost to resolve the ability's effect.
- 7.2.3 As long as the cost changes the game state, a player can use an action ability even if its effect wouldn't change the game state.
- 7.2.4 **Epic Action**: A type of action ability that can only be used **once per game.** After use, place an epic action counter on it. An Epic Action may be used even if the effect wouldn't change the game state, because changing the ability to "used" changes game state. An Epic Action remains used even if the card leaves and re-enters play.
- 7.2.5 Some non-Epic action abilities may only be used a certain number of times per round, phase, or game. A player may use a use-restricted ability even if the effect wouldn't change the game state, because changing the "used" status changes game state.

## 7.3 Constant Abilities

- 7.3.1 A constant ability is **always in effect** while the card it's on is in play. No special formatting — just the text.
- 7.3.2 A constant ability immediately comes into effect when the card enters play and remains in effect while in play.
- 7.3.3 Some constant abilities continuously check a condition (usually containing "while"). The ability's effect only applies while all conditions are true.
  - *Example: Vigilant Honor Guards — "While this unit is undamaged, it gains Sentinel." The ability text is always present, but Sentinel only applies if the unit has no damage.*
  - a. If two conditional constant abilities only satisfy each other's conditions, neither is considered to be met.
- 7.3.4 Constant abilities remain in effect even if the card is exhausted.

## 7.4 Event Abilities

- 7.4.1 An event ability is in the text box of an event and resolves when the event is played.
- 7.4.2 When an event is played, it's placed in its owner's discard pile before its ability resolves. Events move directly from hand to discard pile (a player may place the event in front of them while resolving, but it's in the discard pile for rules purposes).
- 7.4.3 Events that affect cards in a discard pile can affect themselves.
- 7.4.4 Resolve as much of an event ability as possible; ignore anything that can't resolve. A player may pay the cost of an event even if none of the ability would change the game state.
- 7.4.5 An event is both "played" and "resolved" during the action it's played, even if the event ability creates a lasting or delayed effect resolved later.

## 7.6 Triggered Abilities

- 7.6.1 Triggered abilities have **bold triggering condition text** starting with "When" or "On," followed by a colon and an effect. Examples: "When Played," "When Defeated," "On Attack."
- 7.6.2 If a triggered ability has forward slashes ('/') separating multiple triggering conditions, the ability triggers for each condition.
- 7.6.3 For a triggered ability to resolve, the card must be in play when the triggering condition occurs, unless the ability specifies triggering from an out-of-play zone. The triggered ability must still resolve even if the card leaves play before the ability resolves.
  - a. If a card's ability triggers from a hidden zone, the card must be shown to all opponents when it triggers to confirm the trigger exists.
- 7.6.4 Triggered abilities on a unit trigger even if that unit is defeated by the same action that triggered the ability (they resolve after the action finishes).
- 7.6.5 A triggered ability is considered to resolve during the same turn or game step it was triggered.
- 7.6.6 Resolving a triggered ability is NOT the same as taking an action. A player can resolve triggered abilities outside of their turn.
- 7.6.7 A triggered ability **must** resolve once triggered unless it uses "you may." Must resolve at the next available opportunity.
- 7.6.8 If an ability triggers during or as a result of a non-attack action, resolve it at the next available opportunity after the action fully completes. Resolving a triggered ability never interrupts an action or ability that is currently resolving (except during specified attack timing points).
  - a. If an ability instructs a player to perform a modified action, resolve that action and any nested triggers before resolving any part of that ability that specifies it occurs "after" completing that action.
- 7.6.9 If a player must resolve multiple triggered abilities on cards they control at the same time, that player chooses the order.
- 7.6.10 If both players must resolve triggered abilities at the same time, the active player chooses which player resolves first. When chosen, that player resolves all their triggered abilities in the order of their choice, then the other player does the same.
- 7.6.11 After resolving triggered ability "A," any new abilities triggered while resolving it are **nested abilities** and must be resolved before any other abilities triggered at the same time as "A."
- 7.6.12 Any abilities triggered while resolving a nested ability are likewise nested. Each new layer must be fully resolved before returning to an earlier layer.

### 7.6.13 When Played

- a. Triggers when the card is played. Resolves after costs are paid and the card is put into play.
- b. "When Played" abilities, Ambush, and Shielded all resolve in the same timing window, in the order the card's controller chooses.
- c. Any triggered ability whose condition begins with "When played" is a "When Played" ability. Keywords that resolve like "When Played" abilities are NOT "When Played" abilities.

### 7.6.14 When Defeated

- a. Triggers when the card is defeated. Resolves after the card is removed from play and placed in its owner's discard pile.
- b. A card's "When Defeated" ability is resolved by the player who controlled it when it was defeated.
- c. Any ability whose condition begins with "When defeated" is a "When Defeated" ability. Keywords that resolve like "When Defeated" abilities are NOT "When Defeated" abilities.

### 7.6.15 On Attack / On Defense

- a. "On Attack" abilities are triggered when the unit attacks, before combat damage is dealt. "On Defense" abilities trigger when the unit is attacked, before combat damage. Both trigger simultaneously.
- b. Damage from "On Attack" or "On Defense" abilities is NOT combat damage.
- c. "On Attack," "On Defense," "When a unit attacks," and "When a unit is attacked" abilities all resolve in the same timing window.

### 7.6.16 When Attack Ends

- a. Triggers when combat damage is dealt at the final step of an attack.
- b. "When Attack Ends" abilities and "When Defeated" abilities of units defeated by combat damage resolve in the same timing window.
- c. "When Attack Ends" abilities still trigger if the unit they're on is defeated by combat damage.
- d. "While attacking" and "while defending" abilities are no longer active when "When Attack Ends" abilities resolve.
- e. By default, conditions in "When Attack Ends" abilities refer to the attack which caused the trigger.

### 7.6.17 Out-of-Normal-Conditions Use

If a player is instructed to use a triggered ability outside its normal trigger (e.g., using a "When Defeated" ability while the unit is still in play), resolve as much as possible and ignore irrelevant parts.

## 7.7 Effects

- 7.7.1 An effect is a non-cost part of an ability that has the potential to change the game state.
- 7.7.2 Three effect types requiring additional rules: **lasting effects, delayed effects, and replacement effects.**

### 7.7.3 Lasting Effects

- a. A lasting effect is a part of an ability that affects the game for a specified duration. Usually includes "for this phase" or "for this attack."
  - *Examples: "It gets +3/+0 for this attack" (Surprise Strike); "Bases can't be healed for this phase" (Wolffe)*
- b. A lasting effect persists beyond the resolution of the ability that created it, even if the card left play.
- c. Multiple lasting effects can apply simultaneously. A new lasting effect that conflicts with an existing one takes precedence.
- d. By default, a lasting effect only applies to cards in play at the time it was created.
- e. Lasting effects expire immediately at the end of their specified duration. All effects that expire at the same timing window do so simultaneously.

### 7.7.4 Delayed Effects

- a. A delayed effect is created when an ability indicates a future timing point or condition and an effect at that time.
  - *Examples: "At the start of the regroup phase, draw 1 card"; "The next card you play this phase costs 1 less"*
- b. Delayed effects resolve automatically and immediately after their specified timing point/condition occurs, before any other abilities triggered by that timing.
- c. Once created, a delayed effect resolves at its specified point/condition, even if the ability's card left play.
- d. A resolving delayed effect is not treated as a new triggered ability.
- e. If multiple delayed effects must resolve at the same time, the same rules apply as for simultaneous triggered abilities.
- f. If the expiration timing/condition has already occurred when the effect is created, it resolves immediately.

### 7.7.5 Replacement Effects (Instead, Would)

- a. A replacement effect replaces part or all of the standard resolution of a triggering condition, ability, or action step with an alternate resolution.
  - *Examples: Luke Skywalker: Jedi Knight — normally gives -3/-3 for the phase; if a friendly unit was defeated this phase, gives -6/-6 instead.*
  - *Shield token — normally damage is dealt; instead, prevent the damage and defeat the Shield.*
- b. Indicated by the words **"instead"** or **"would."**
- c. A replacement effect must be resolved immediately when its condition is met, unless it uses "you may." If the player cannot perform the replacement, they must resolve the original effect.
- d. If a replacement effect replaces ALL of the standard resolution, the standard resolution doesn't resolve and only the replacement can trigger abilities.
- e. If multiple replacement effects are triggered by the same condition, the player who controls the affected game object chooses the order to resolve them. Once an effect has been replaced, other pending effects can't replace that already-replaced effect.
- f. A replacement effect cannot replace itself.
