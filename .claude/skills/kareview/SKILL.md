# kareview

Review a card implementation and test spec for correctness, completeness, and PR readiness using parallel agents.

## Step 0: Identify the card and files

From the user's message, identify:
- **Implementation file:** `server/game/cards/[SET]/[type]/[ClassName].ts`
- **Test spec:** `test/server/cards/[SET]/[type]/[ClassName].spec.ts`
- **Card's printed ability text** — ask if not provided (required for accuracy check)

Check `test/json/Card/` for the card's JSON data file. If the directory is empty or the card file is missing, tell the user:
> "Card data not found. Please run `npm run get-cards` first, then try again."

Note: `test/json/Card/` files are auto-generated by `npm run get-cards`. The `"text"` field always reads `"mock ability text"` for mock cards — this is expected and is NOT a bug or issue to flag.

Read the implementation file and spec file before launching agents.

## Step 1: Pre-flight checks

Run directly (do not delegate) — report immediately if any fail:

1. **TypeScript structure:** Correct imports, base class, `getImplementationId`, `setupCardAbilities` signature present.
2. **Card registration:** `getImplementationId()` `id` matches the `id` field in `test/json/Card/[internalName].json`. Mismatch = card won't load.
3. **Test file exists:** No spec = blocker, not PR ready.

## Step 2: Launch 3 parallel review agents

**CRITICAL:** All 3 in a **single message**, all `subagent_type: "Explore"`.

---

**Agent A — Implementation review**

Read [IMPL_FILE_PATH]. Card text: [CARD TEXT].

Classify each as **OK / WARN / ERROR**:
1. **Base class** — `NonLeaderUnitCard`, `LeaderUnitCard`, `EventCard`, `UpgradeCard`, `BaseCard`?
2. **Registrar method** — matches trigger? ("When Played" → `addWhenPlayedAbility`, "On Attack" → `addOnAttackAbility`, passive → `addConstantAbility`, etc.)
3. **Ability properties** — `title` reflects text; `optional: true` only if "you may"; `when:` correct; `limit` if "once per round/phase"; `cost` if activation cost
4. **immediateEffects** — correct functions and amounts?
5. **ongoingEffects** — `modifyStats`, `gainKeyword`, `gainAbility` used correctly if constant ability?
6. **State watchers** — `setupStateWatchers` overridden and watcher queried correctly if phase/round history needed?
7. **Targeting** — `cardTypeFilter`, `controller`, `zoneFilter`, `cardCondition` match card text restrictions?
8. **TypeScript imports** — all needed imports present?

---

**Agent B — Test spec review**

Read [SPEC_FILE_PATH].

**If the spec file does not exist:** Report "NO SPEC — not PR ready." Then generate a complete, working test skeleton:
- `describe → integration(contextRef) → describe → beforeEach → it()` structure
- Happy path `it()`: trigger ability, `toBeAbleToSelectExactly` if targeting, assert board state, `toBeActivePlayer()`
- Edge case `it()`: most likely failure mode (friendly unit unaffected, no valid targets, optional skip)
- Use real card names from codebase (`wampa`, `tieln-fighter`, `battlefield-marine`) as test opponents
- Reference `docs/testing-cards.md` and `docs/test-cheat-sheet.md`

**If spec exists**, classify each as **OK / WARN / ERROR**:
1. **Structure** — `describe` → `integration` → `describe` → `beforeEach` → `it()`?
2. **Test setup** — card in correct zone; opponent cards for targeting; preconditions set?
3. **Happy path** — ability triggers fully, correct board state verified?
4. **Edge cases** — condition not met; optional pass; `toBeAbleToSelectExactly`; limit tested?
5. **Assertion quality** — `toBeInZone`, `.damage`, `toHavePrompt`, `toBeActivePlayer` used appropriately?
6. **Test isolation** — each `it()` starts from `beforeEach` state?
7. **PR readiness** — obvious missing tests a reviewer would ask for?

---

**Agent C — Card text accuracy**

Read [IMPL_FILE_PATH] and [CARD_DATA_PATH]. Card text: [CARD TEXT].

Classify each as **ACCURATE / DISCREPANCY / MISSING**:
1. **Trigger condition** — registrar method + `when:` matches printed trigger?
2. **Effect accuracy** — correct functions, amounts, targets?
3. **Targeting restrictions** — `cardTypeFilter`, `controller`, `zoneFilter` match printed restrictions?
4. **Optional vs mandatory** — "you may" → `optional: true`; no "you may" → not optional?
5. **Limitations/costs** — "once per round" → `limit`; activation cost → `cost` array?
6. **Keywords** — keywords granted by ability implemented via `gainKeyword`?
7. **Missing abilities** — any printed text unimplemented?
